## 1. 시스템 개요

### 1.1 목적
상담사 팀의 업무 효율화를 위한 인트라넷 기반 CRM(고객 관계 관리) 시스템.  
할일 배분, KPI 관리, 팀 내 실시간 채팅, 프로모션 정보 공유 등 업무 전반을 하나의 플랫폼에서 처리한다.

### 1.2 기술 스택

| 항목 | 내용 |
|------|------|
| Backend | Python / Flask 3.0 |
| 실시간 통신 | Flask-SocketIO 5.3 (Socket.IO) |
| Database | PostgreSQL |
| Server | Gunicorn 21.2 + Eventlet (포트 5001, 5002 이중화) |
| Frontend | Vanilla JavaScript + Socket.IO Client |
| 스타일 | 커스텀 CSS + 다크모드 지원 |
| PWA | Service Worker + Web Push 알림 |

### 1.3 접속 환경
- 내부망: 전체 기능 사용
- 외부망: HTTPS 접속 (nginx 프록시)
- 모바일: 반응형 웹, PWA 홈화면 추가 지원

## 3. 로그인 / 인증

### 3.1 로그인

- **URL:** `GET/POST /login`
- **입력:** 아이디, 비밀번호
- **처리:**
  - bcrypt 해시 비교
  - 실패 5회 시 1시간 잠금 (Rate Limiting)
  - 비활성 계정은 로그인 거부
- **세션:** 1시간 유지, HTTPS 전용 쿠키 (Secure, HttpOnly, SameSite=Lax)

### 3.2 초기 비밀번호

| 역할 | 초기 비밀번호 |
|------|--------------|
| 관리자 | `admin1234` |
| 상담사/팀장 | `body123!` |

### 3.3 비밀번호 변경
- **URL:** `POST /api/change-password`
- **Rate Limit:** 분당 3회
- **검증:** 현재 비밀번호 확인 후 변경

## 4. 할일 관리 (메인)

### 4.1 화면 구성
- **URL:** `GET /`
- 업무 카드 목록 (상태별 탭: 전체 / 대기중 / 진행중 / 완료)
- 상단 통계 배지 (미완료, 진행중 건수)

### 4.2 업무 상태 흐름

```
대기중 → 진행중 → 완료
```

- 상담사: 자신에게 할당된 업무만 조회 / 상태 변경 가능
- 관리자/팀장: 전체 업무 조회 / 생성 / 삭제 / 배분

### 4.3 업무 생성

**입력 항목:**
| 항목 | 필수 | 설명 |
|------|------|------|
| 제목 | O | 업무 제목 |
| 내용 | X | 상세 내용 |
| 팀 | X | 특정 팀 지정 (미지정 시 전체) |

### 4.4 업무 배분

#### 4.4.1 개별 배분
- 특정 업무를 특정 사용자에게 직접 배분

#### 4.4.2 일괄 배분 (Bulk Assign)
- 미배분 업무 전체를 대상으로 배분
- **개별:** 선택한 사용자 한 명에게 전체 배분
- **랜덤:** 활성 상담사 중 무작위 1명에게 배분
- **순차:** 팀 / 입사일 순서 기준으로 한 명씩 순환 배분

#### 4.4.3 엑셀 일괄 등록
- 템플릿 다운로드: `GET /download/template/tasks`
- 업로드: `POST /api/items/bulk-upload`
- 검증: 컬럼 구조 확인 후 일괄 삽입

### 4.5 업무 수정 / 삭제
- 관리자/팀장만 가능
- 수정: 제목, 내용, 팀 변경
- 삭제: 완료 상태 여부 무관 삭제 가능

## 5. 실시간 채팅

### 5.1 채팅 목록

- **URL:** `GET /chats`
- 참여 중인 채팅방 목록
- 각 채팅방: 마지막 메시지, 읽지 않은 메시지 수 배지
- 관리자: 전체 채팅방 목록 접근 가능

### 5.2 채팅방 생성

- **URL:** `GET /chat/create`
- 제목 입력 + 참여자 선택 (사용자 검색 자동완성)
- 1:1 또는 그룹 채팅

### 5.3 채팅방

- **URL:** `GET /chat/<chat_id>`
- 참여자만 입장 가능 (관리자는 참여 여부 무관 접근 가능)

#### 5.3.1 메시지 전송
- 텍스트, 이미지, 파일 첨부 (최대 50MB)
- 클립보드 이미지 붙여넣기 지원 (Ctrl+V)
- 링크 자동 미리보기

#### 5.3.2 읽음 표시
- 카카오톡 스타일: 각 메시지 옆에 읽지 않은 인원 수 표시
- 채팅방 입장 시 자동으로 모든 메시지 읽음 처리

#### 5.3.3 타이핑 인디케이터
- 상대방이 입력 중일 때 "..." 표시

#### 5.3.4 메시지 검색
- 채팅방 내 텍스트 검색
- 날짜 기준 이동 (날짜 피커)
- 검색 결과 클릭 시 해당 메시지로 이동 (전후 맥락 포함)

#### 5.3.5 채팅방 설정
- 제목 변경
- 참여자 추가 / 제거
- 알림 음소거 토글
- 채팅방 나가기

#### 5.3.6 민감정보 필터
- 전화번호, 계좌번호 등 민감 패턴 자동 필터링

### 5.4 실시간 이벤트 (Socket.IO)

| 이벤트 | 방향 | 설명 |
|--------|------|------|
| join | 클→서 | 채팅방 입장 |
| send_message | 클→서→클 | 메시지 전송 및 브로드캐스트 |
| typing_start | 클→서→클 | 타이핑 시작 알림 |
| typing_stop | 클→서→클 | 타이핑 중지 알림 |
| mark_as_read | 클→서 | 읽음 처리 |
| nav_counts_update | 서→클 | 배지 수량 갱신 |

## 7. KPI 점수 관리

### 7.1 화면 구성

- **URL:** `GET /kpi` (관리자 / 팀장 전용)
- 좌측: 카테고리 목록
- 우측: 상담사별 카테고리 점수 테이블 (월별 조회)

### 7.2 조회 기준
- 년도 / 월 선택 (이전/다음 버튼)
- 팀별 구분 표시 (팀 정렬 순서 적용)
- 상담사 정렬: 팀 순서 → 입사일 오름차순

### 7.3 노출 대상 규칙

| 상태 | 조건 | 결과 |
|------|------|------|
| 활성 상담사 | role = '상담사', status = 'active' | 항상 표시 |
| 비활성 상담사 | status = 'inactive' | 비활성화된 월까지만 표시 |
| 관리자 승격된 상담사 | role = '관리자', promoted_to_admin_date 존재 | 승격된 월까지만 표시 |

**예시:**
- 3월 10일 비활성 전환 → 3월 KPI에 표시, 4월부터 미표시
- 3월 15일 관리자 승격 → 3월 KPI에 표시, 4월부터 미표시

### 7.4 점수 입력

- 카테고리 클릭 → 해당 상담사의 카테고리 점수 입력 모달
- 날짜별 점수 기록 (하나의 카테고리에 여러 날짜 입력 가능)
- 메모(비고) 입력 가능

### 7.5 카테고리 관리

- 카테고리 추가 / 수정 / 삭제 (soft delete - 기존 점수 보존)
- 드래그로 순서 변경

### 7.6 변환 공식

- 원시 점수를 공식에 따라 환산 점수로 자동 계산
- 공식 활성화 / 비활성화 토글
- 공식 복사 기능

### 7.7 엑셀 내보내기

- `GET /api/kpi/export?year=&month=`
- 시트 구성: 팀별 상담사 × 카테고리 점수 + 합계

## 10. 운세 & 타로 & 로또

### 10.1 화면 구성

- **URL:** `GET /fortune`
- 탭: 오늘의 운세 / 타로카드 / 로또 생성기

---

### 10.2 오늘의 운세

**입력:**
| 항목 | 필수 | 설명 |
|------|------|------|
| 이름 | O | 사용자 이름 |
| 생년월일 | O | YYYY-MM-DD |
| 조회 날짜 | X | 기본값: 오늘 |

**처리:**
- `fortune.py` 엔진에서 생년월일 기반 운세 계산
- 날짜별 고유 운세 결과 생성

**결과:**
- 오늘의 운세 텍스트
- 운수 수치 (행운, 재물, 건강, 사랑 등)
- 오늘의 조언

---

### 10.3 타로카드 리딩

**스프레드 유형:**

| 유형 | 카드 수 | 설명 |
|------|---------|------|
| one_card | 1장 | 단순 오늘의 메시지 |
| three_card | 3장 | 과거 / 현재 / 미래 |
| love_spread | 3장 | 연애 특화 |
| yes_no | 1장 | Yes / No 판단 |

**입력:**
- 스프레드 유형 선택
- 질문 텍스트 (선택)
- 카드 선택 (무작위 또는 직접 선택)

**처리:**
- `tarot.py` 엔진에서 카드 의미 및 해석 생성
- 정방향 / 역방향 랜덤 적용

**결과:**
- 카드 이미지 / 이름 / 원소 / 키워드
- 카드 상세 의미 (정방향 / 역방향)
- 질문에 대한 종합 해석

**카드 구성:**
- 메이저 아르카나 22장 (0번 광대 ~ 21번 세계)
- 각 카드별 심볼, 원소, 키워드 정보 포함

---

### 10.4 로또 번호 생성기

**면책 문구:** "도박은 해롭습니다. 단순 재미용이며 결과에 책임지지 않습니다."

**입력:**
- 게임 수 선택: 1 ~ 5게임

**처리:**
- Fisher-Yates 셔플 알고리즘으로 1~45 중 7개 추출
- 6개 메인 번호 + 1개 보너스 번호
- 게임 수만큼 반복 생성

**번호 색상 기준:**

| 범위 | 색상 |
|------|------|
| 1 ~ 10 | 노란색 |
| 11 ~ 20 | 파란색 |
| 21 ~ 30 | 빨간색 |
| 31 ~ 40 | 회색 |
| 41 ~ 45 | 초록색 |

**결과 표시:**
- 1게임: 메인 볼 6개 + 보너스 볼 1개 개별 표시
- 2~5게임: A~E 행으로 구성된 표 형태
- 볼 드롭 애니메이션 효과 (ballDrop 키프레임)
- 재미 메시지 랜덤 표시 (당첨 기원 문구)

### 11.2 사용자 생성

|------|------|

### 11.5 입사일 표시 조건

- 사용자 관리 화면에서 조회 및 수정 가능
